Crea un nuevo componente React para este proyecto de gestión de turnos de psicología.

Descripción del componente: $ARGUMENTS

Requisitos:
- Archivo en `src/components/<NombreComponente>.tsx`.
- Si tiene estado o efectos → `"use client"` al inicio.
- Si es solo presentación → Server Component (sin directiva).
- Clases Tailwind del sistema de diseño: `card`, `btn-primary`, `btn-ghost`, `btn-danger`, `chip`, `input`, `label`, `textarea`, `section-title`.
- Tipado TypeScript estricto con interfaces/types definidas en el mismo archivo.
- Iconos de `lucide-react`.
- Si hace fetch → usar `fetch("/api/...")` con manejo de error básico.
- Sin abstracciones extra ni props opcionales que no se pidan.

Generá solo el componente pedido, sin wrappers innecesarios.
